Struktura Schematu Centralnej Bazy Materiałowej

Dane w pliku baza_materialow.js są zapisane w formie tablicy obiektów JavaScript przypisanej do zmiennej globalnej MOJA_BAZA_MATERIALOW. Każdy obiekt musi ściśle przestrzegać następującej struktury kluczy:

window.MOJA_BAZA_MATERIALOW = [
  {
    "name": "Klucz płasko-oczkowy 13mm", // Ciąg znaków wyświetlany w polu wyboru (Unikalny)
    "kat": "Narzędzia ręczne",        // Flaga kategoryzacji (Musi odpowiadać data-kat przycisków)
    "uwagi": "Knipex / Stanley"        // Wartość automatycznie wstrzykiwana do kolumny Uwagi
  }
];

2. Implementacja Mechanizmu Dual-Logic

Aby umożliwić jednoczesne filtrowanie wykazu sprzętu oraz masowe ładowanie danych przy pomocy jednego zestawu kontrolek, wdrożono algorytm rozróżniania zdarzeń oparty na asynchronicznym opóźnieniu czasowym (Debounce/Timer).

Mechanizm rejestracji zdarzeń w pętli DOM

Przyciskom przypisana jest klasa .btn-pkg oraz atrybut data-kat. Skrypt JavaScript przypisuje dwa niezależne nasłuchiwania (EventListener) na każdy element:

  1. Zdarzenie pojedynczego kliknięcia ('click'):
    • W momencie wywołania sprawdzane jest czy zmienna clickTimer przechowuje referencję do aktywnego licznika. Jeśli tak, zostaje ona natychmiast zresetowana za pomocą clearTimeout(clickTimer).
    • Inicjalizowany jest nowy licznik setTimeout z czasem oczekiwania wynoszącym dokładnie 250ms.
    • Jeśli w tym interwale nie nastąpi drugie kliknięcie, skrypt wykonuje operację filtrowania: ustawia zmienną AKTUALNIE_AKTYWNA_KATEGORIA na wartość pobraną z data-kat, nadaje klasę .active klikniętemu przyciskowi i wywołuje funkcję przebudujDatalistPodKategorie().
  2. Zdarzenie podwójnego kliknięcia ('dblclick'):
    • Wywoływane, gdy przeglądarka zarejestruje drugie kliknięcie w czasie krótszym niż 250ms.
    • Pierwszym krokiem jest bezwzględne zniszczenie oczekującego licznika: clearTimeout(clickTimer), co całkowicie zapobiega uruchomieniu procedury pojedynczego kliknięcia.
    • Następuje wymuszenie podświetlenia przycisku (klasa .active) oraz bezpośrednie wywołanie funkcję masowego, inteligentnego wstrzykiwania: wczytajMaseSprzetuZFilmu(AKTUALNIE_AKTYWNA_KATEGORIA).

Specyfikacja systemu pomocy (Korekta UI)

Opisy stanów interfejsu w pliku uruchomieniowym zostały sformalizowane językowo w celu ujednolicenia pojęć. Wykluczono potoczne określenia (np. "branża"), zastępując je standaryzacją techniczną:

  • Pojedyncze kliknięcie: Blokuje obiekt <datalist> w kolumnie "Nazwa materiału" na wybranej kategorii.
  • Podwójne kliknięcie (Szybki dwuklik): Wstrzykuje kompletny pakiet pozycji z wybranej kategorii do pustych węzłów tabeli DOM.

3. Walidacja Duplikatów i Kontrola Stanu Wierszy

Algorytm dynamicznego usuwania pozycji z wykazu sprzętu

Aby wykluczyć możliwość dwukrotnego wpisania tego samego przedmiotu (co generuje problemy formalne podczas rewizji materiałowej), element <datalist id="slownik-materialow"> jest w całości czyszczony i generowany od zera przy każdej modyfikacji tabeli.

Funkcja przebudujDatalistPodKategorie() działa według ściśle określonego schematu:

  1. Iteruje przez wszystkie wiersze tabeli od 1 do 55 i pobiera ciągi tekstowe z pól input-nazwa-[i]. Jeśli pole nie jest puste, jego zawartość jest oczyszczana metodą trim().toLowerCase() i zapisywana w tymczasowej tablicy aktualnieUzyteNazwy.
  2. Następuje odczyt globalnej tablicy wykazu sprzętu MOJA_BAZA_MATERIALOW.
  3. W pętli sprawdzane są dwa warunki:
    if ((KategoriaPrzedmiotu == KategoriaAktywna || KategoriaAktywna == "Wszystkie") && !aktualnieUzyteNazwy.includes(NazwaZeSpisu.toLowerCase()))
  4. Tylko pozycje spełniające ten warunek są wstrzykiwane jako tagi <option> do listy podpowiedzi.

Mechanizm detekcji zmian i style CSS kontrastu wysoki/niski

Wprowadzenie tekstu do dowolnego pola wiersza aktywuje funkcję obslugaZmianyInputu(index). Wykonuje ona natychmiastowe sprawdzenie logiczne:

  • Jeśli pola wiersza zawierają jakiekolwiek znaki, do elementu <tr id="row-tr-[index]"> dopisywana jest klasa CSS .has-data.
  • Klasa .has-data wymusza zmianę tła wiersza z szarego na czystą biel (#ffffff !important) oraz zmienia wagę czcionki tekstu na font-weight: 700 (kolor głęboka czerń #000000). Pozwala to na błyskawiczne odróżnienie pozycji wypełnionych od pustych.
  • Wyczyszczenie wiersza przyciskiem kosza 🗑️ (funkcja clearRowData(index)) usuwa klasę .has-data, co natychmiastowo przywraca szary odcień tła i oddaje usunięty przedmiot z powrotem do bazy podpowiedzi. Oczyszczono nagłówek z redundantnych sformułowań (usunięto frazę "dla ochrony").

Logika dynamicznej aktywacji numeracji LP (Wiersze 31 - 55)

Dla ochrony geometrii dokumentu i czytelności, wiersze od 31 do 55 mają pierwotnie ukrytą numerację (wyświetlają znak — za pomocą klasy .empty-lp). Dopiero wywołanie zdarzenia onfocus w polu nazwy danego wiersza uruchamia funkcję activateRow(index), która trwale podmienia tekst komórki na realną cyfrę indeksu, zatwierdzając wiersz do fizycznego ujęcia na wydruku. Nagłówek sekcji zmapowano jako Ukrywanie wpisanych pozycji.

4. Obsługa Obiektów JSON i Architektura Wydruku PDF

Struktura Pliku KpII Szablonu (.json)

Funkcja exportTemplateJSON() konsoliduje stan formularza i buduje zunifikowany obiekt danych, który jest pobierany na dysk jako plik tekstowy. Struktura danych została zaprojektowana w sposób płaski w celu zachowania maksymalnej profesjonalnej szybkości parsowania:

{
  "przepustka": "123/2026",
  "wykonawca": "HVAC Master Serwis",
  "data": "2026-06-17",
  "kierunek": "wwoz", // Akceptowalne wartości: 'wwoz' lub 'wywoz'
  "nazwisko": "Kowalski Jan",
  "pesel": "850512XXXXX",
  "marka": "Ford S-Max",
  "rejestracja": "GKW 12345",
  "items": [            // Tablica zawierająca dokładnie 55 obiektów pozycji
    { "lp_text": "1", "nazwa": "Młotek", "jm": "szt.", "qty": "1", "uwagi": "Stanley" },
    { "lp_text": "—", "nazwa": "", "jm": "", "qty": "", "uwagi": "" }
  ]
}

Logika Generowania Stron PDF w onbeforeprint

Podczas wywołania systemowego okna drukowania, funkcja window.onbeforeprint() przeprowadza automatyczną kalkulację rozkładu pozycji na fizycznych stronach formatu A4:

Zakres wierszy edycji Strona docelowa PDF Warunek aktywacji sekcji
Wiersze 1 - 5 Strona 1 (Główna) Zawsze aktywna. Zawiera pełny blok metadanych pracownika, pojazdu, klauzulę RODO oraz sekcje podpisów i kontroli ochrony.
Wiersze 6 - 30 Strona 2 (Kontynuacja) Aktywowana (display: block), jeśli choć jeden wiersz z tego przedziału posiada przypisaną klasę .has-data. W przeciwnym wypadku obiekt jest ukrywany, a łączna liczba stron w nagłówku ustawia się na 1.
Wiersze 31 - 55 Strona 3 (Dodatkowa) Aktywowana, gdy pozycje wykroczą poza indeks 30. Wymusza ustawienie łącznej liczby stron na 3. Modyfikuje widoki liczników stron na wszystkich nagłówkach papierowych dokumentu.

5. Zatwierdzony Plan Rozwoju (Backlog Techniczny)

Poniższa sekcja zawiera wytyczne dla przyszłych modyfikacji kodu, ustalone na podstawie analizy ergonomii pracy terenowej:

📌 ZADANIE: Automatyzacja pól J.m. oraz Ilości ("Klikaj mniej")

Status: Oczekuje na implementację.

Cel algorytmu: Całkowite odciążenie użytkownika od konieczności ręcznego przeklikiwania kolumn pobocznych przy standardowym wyposażeniu.

Wytyczne techniczne: Należy zmodyfikować funkcje obslugaZmianyInputu(index) oraz mechanizm pętli wczytajMaseSprzetuZFilmu(). W momencie, gdy pole nazwy narzędzia zostaje uzupełnione wartością niepustą, program ma wykonać test: jeśli pola Jednostki miary oraz Ilości są puste, skrypt ma bezpośrednio przypisać do właściwości .value pól input odpowiednio ciąg znaków "szt." oraz cyfrę "1". Modyfikacja musi automatycznie nadać wierszowi klasę .has-data.

🟢 STATUS: Zrealizowano pomyślnie (Wdrożono do Baza-Edytor.html)

Mechanizm kontroli literówek i licznika nowości: Zaprojektowano i wdrożono algorytm porównujący asynchronicznie zawartość zewnętrznych plików JSON z lokalną strukturą bazy danych. System poprawnie zlicza unikalne elementy, filtruje duble metodą toLowerCase() i uruchamia modalny bufor weryfikacyjny. Funkcja obsługuje bezpieczne modyfikacje wartości w locie przed zatwierdzeniem oraz masowe wypychanie pozycji na szczyt tablicy metodą unshift/prepend z przypisaniem klasy wizualnej .is-new-row.

Decyzje projektowe (Odrzucone/Zamknięte):

  • Blokada przechowywania danych w LocalStorage: Ze względów bezpieczeństwa (praca na losowych komputerach, ochrona PESEL oraz danych rejestracyjnych pojazdów) zabrania się implementacji automatycznego zapisu profilu w pamięci podatnej przeglądarki. Dane mogą być przenoszone wyłącznie za pomocą bezpiecznych, jawnych plików JSON kontrolowanych przez użytkownika.
  • Brak dedykowanego przycisku Reset: Funkcja czyszczenia całego formularza została powierzona natywnemu odświeżeniu strony za pomocą klawisza F5. Pozwala to uniknąć wprowadzania zbędnych skryptów czyszczących tablice i upraszcza architekturę kodu interfejsu.
